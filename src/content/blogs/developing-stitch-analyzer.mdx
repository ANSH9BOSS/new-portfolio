# Developing Stitch Analyzer: Anti-Cheat Packet Interception

Security is a primary pillar of game infrastructure. Traditional anti-cheat engines are easily bypassed by modern client injects and hardware macro setups. To combat this, I designed and built **Stitch Analyzer**—a dual-sided scanning framework.

## The Dual-Sided Scan Concept

Relying solely on client memory checks is dangerous because modern inject loaders execute inside target applications without modifying native files on disk. Relying solely on server-side checks is also limiting because high latency can cause false flag detections.

Stitch Analyzer solves this by pairing:
1. **Client-Side Scanner**: Built on Electron, performing process memory scans, signature validation, and signing runtime packets.
2. **Server-Side Interceptor**: Built on Java NMS (Netty handlers) to intercept incoming packets.

## Java NMS Packet Hooking

To capture client combat inputs, click patterns, and movement packets, we injected a custom pipeline handler into Netty:

```java
public class PacketHandler extends ChannelDuplexHandler {
    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
        if (msg instanceof PacketPlayInFlying) {
            // Analyze velocity vectors and coordinate offsets
        }
        super.channelRead(ctx, msg);
    }
}
```

These packets are decrypted, validated against cryptographic signatures generated by the client scan helper, and analyzed in real-time.

## Conclusion

Combining cryptographic client-state verification and low-level server NMS packet intercepts reduced hack client injection rates to zero.
