# Building Stitch Hosting: Automating Pterodactyl VPS Containers

Scaling a Minecraft and VPS hosting provider requires more than just high-end CPU cores; it requires seamless virtualization orchestration and billing automation. In this article, I discuss how we built **Stitch Hosting** using WHMCS billing APIs and Pterodactyl container nodes.

## The Architectural Challenge

When a client purchases a server, they expect it to be active within seconds. Manually creating virtualization slots on nodes is slow, error-prone, and doesn't scale.

We needed a system that could:
1. Capture client transaction triggers on WHMCS checkout.
2. Verify payment status and select an optimal physical host node (based on CPU/memory loads).
3. Connect with Pterodactyl’s daemon API (Wings) to spin up isolated container instances dynamically.

## Integrating WHMCS with Pterodactyl API

We wrote a custom integration handler that hooks into WHMCS checkout events. When a payment validation webhook is received:

- A script query fetches node capacity details.
- A REST payload is transmitted to the Pterodactyl control panel node:
  ```json
  {
    "name": "Client Server",
    "user": 14,
    "egg": 3,
    "docker_image": "ghcr.io/pterodactyl/yolks:java_17",
    "limits": {
      "memory": 8192,
      "swap": 0,
      "disk": 20480,
      "io": 500,
      "cpu": 200
    }
  }
  ```
- Pterodactyl immediately orchestrates the Docker container deployment, emails client credentials, and registers port forwarding.

## Conclusion

Automated provisioning transformed our hosting workflow, reducing server setup times to under 10 seconds.
